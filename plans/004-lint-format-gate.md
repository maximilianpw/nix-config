## Why this matters

alejandra only runs inside `scripts/nixos-rebuild.sh` (line 172-177), so a
plain `git commit` can land unformatted Nix, and nothing in the repo ever runs
a Nix linter: `statix` (anti-patterns like `if x then true else false`,
manual attrs where `inherit` belongs) and `deadnix` (unused lambda args and
let-bindings) appear nowhere. This plan adds one lint derivation to the flake
`checks`, a `make lint` target, a CI job, and a git pre-commit hook via
`git-hooks.nix` so violations are caught at commit time. The repo is plain git
(no `.jj/` — verified), so git hooks fire on every commit.

## Steps

### Step 1: Baseline — run the linters and record findings

```sh
nix run nixpkgs#statix -- check . 2>&1 | tee /tmp/statix-baseline.txt
nix run nixpkgs#deadnix -- --fail --exclude machines/hardware -- . 2>&1 | tee /tmp/deadnix-baseline.txt
```

**Verify**: both commands complete (non-zero exit with findings is expected).
Count the findings; if either reports more than ~40 findings, STOP (see STOP
conditions).

### Step 2: Fix the baseline findings

Apply the fixes statix/deadnix suggest, file by file. Rules:
- deadnix "unused lambda arg" in a *module signature* (e.g. `config,` never
  used): remove the arg unless removing it empties the attrset pattern.
- statix suggestions: apply as suggested unless the fix changes evaluation
  semantics you cannot verify — in that case skip and note it.
- Do not fix anything in `machines/hardware/`.

**Verify**: both Step 1 commands now exit 0 with no findings, and
`nix flake check --no-build` → exit 0 (proves fixes didn't break eval).

### Step 3: Add the lint check + pre-commit hook to flake.nix

Add the input (top of inputs, alphabetical-ish placement near disko):

```nix
git-hooks.url = "github:cachix/git-hooks.nix";
git-hooks.inputs.nixpkgs.follows = "nixpkgs";
```

In outputs, add `git-hooks` to the destructured args. Replace the `checks`
attr with a version that keeps the existing eval checks and adds, **per
system** (x86_64-linux and aarch64-darwin):

```nix
pre-commit-check = inputs.git-hooks.lib.${system}.run {
  src = ./.;
  hooks = {
    alejandra.enable = true;
    statix.enable = true;
    statix.settings.ignore = ["machines/hardware"];
    deadnix.enable = true;
    deadnix.settings.exclude = ["machines/hardware"];
  };
};
```

Structure suggestion: introduce a small `let mkChecks = system: {...} in`
helper or write the two system attrsets explicitly — match the file's
existing plain style (it does not use flake-parts; do not introduce it).
Then wire hook installation into the existing devShell (flake.nix:153-162):

```nix
shellHook = ''
  ${self.checks.${system}.pre-commit-check.shellHook or ""}
  echo "Welcome to the Nix dev shell for ${system}"
'';
```

Note: `git-hooks.lib.${system}.run` exposes `.shellHook`, which installs
`.git/hooks/pre-commit` when the devShell is entered.

Run `nix flake lock` and `alejandra flake.nix`.

**Verify**:
1. `nix flake check --no-build` → exit 0
2. `nix build .#checks.<current-system>.pre-commit-check --no-link` → exit 0
3. `nix develop -c true` then `test -f .git/hooks/pre-commit` → exit 0

### Step 4: Add `make lint`

In `Makefile`, following the style of the existing targets, add:

```make
lint: ## Run Nix linters (statix, deadnix) and format check
	nix build .#checks.$$(nix eval --impure --raw --expr builtins.currentSystem).pre-commit-check --no-link
```

(If the Makefile uses a simpler pattern for other targets, match it — e.g. a
hardcoded call to `nix flake check`. Read the Makefile first and mimic.)

**Verify**: `make lint` → exit 0.

### Step 5: Add a CI lint job

In `.github/workflows/ci.yml` add:

```yaml
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: DeterminateSystems/determinate-nix-action@v3
      - run: nix build .#checks.x86_64-linux.pre-commit-check --no-link
```

**Verify**: `nix build .#checks.<current-system>.pre-commit-check --no-link` → exit 0 (same derivation CI will build).
