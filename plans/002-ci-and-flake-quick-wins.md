## Why this matters

Four small independent fixes bundled because each is under 30 minutes:

1. The weekly package-update workflow tries to `nix-update` a package named
   `xurl` that has no definition in `packages/`, so every scheduled run logs a
   hard failure and pollutes the PR body warning, masking real bump failures.
2. CI builds the two Linux systems but never evaluates the darwin config, so a
   change that breaks `macbook-pro-m1` merges green and is only discovered at
   the next local `make rebuild` on the Mac.
3. `.agents/skills/` and `.claude/skills/` are byte-identical copies; every
   skill edit must be made twice and they will silently diverge.
4. The `fenix` flake input (Rust toolchain) follows the *stable* nixpkgs
   channel; fenix upstream expects unstable, so rust-analyzer and toolchain
   components age on the quarterly stable cadence instead of rolling.

### Step 2: Add a darwin eval job to CI

In `.github/workflows/ci.yml`, add a third job after `build-linux`:

```yaml
  eval-darwin:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: DeterminateSystems/determinate-nix-action@v3
      - run: nix eval --raw .#darwinConfigurations.macbook-pro-m1.system.drvPath
```

This is eval-only (no macOS runner needed): it catches module typos, missing
files, and option errors in the darwin config. Do not attempt `nix build` of
the darwin system on Linux.

**Verify**: locally run the same eval command → prints a drv path, exit 0.
Also `nix run nixpkgs#action-validator -- .github/workflows/ci.yml` if
available; otherwise visually confirm YAML indentation matches the existing
jobs (2-space, same structure).

### Step 3: Replace `.agents/skills` with a symlink to `.claude/skills`

```sh
diff -r .agents/skills .claude/skills   # must be identical before deleting
rm -r .agents/skills
ln -s ../.claude/skills .agents/skills
git add .agents/skills
```

Git stores the symlink; both paths keep working for tools that read either
location.

**Verify**: `diff -r .agents/skills/ .claude/skills/` → no output, exit 0; and
`git status --short .agents` shows the symlink change staged.

### Step 4: Point fenix at nixpkgs-unstable

In `flake.nix:18` change `inputs.nixpkgs.follows = "nixpkgs";` to
`inputs.nixpkgs.follows = "nixpkgs-unstable";` inside the `fenix` block, then:

```sh
nix flake update fenix
alejandra flake.nix
```

**Verify**:
1. `nix flake check --no-build` → exit 0
2. `nix eval --raw .#nixosConfigurations.main-pc.config.system.build.toplevel.drvPath` → prints a drv path (this fully evaluates the Linux config including the fenix toolchain in dev-tools.nix). This eval takes a minute or two; that's normal.
